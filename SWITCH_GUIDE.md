## 二、接线步骤

### 第一步：理清设备

```
操作前确认每台设备已贴标签:
  □ CR-1 ~ CR-6 (中兴)
  □ CS 服务器
  □ TS 服务器 (与 CS 同机时跳过)
  □ AP-1 工控机/服务器
  □ AP-2 工控机/服务器
  □ Host-1 笔记本
  □ Host-2 笔记本
  □ 交换机 #1 (管理面)
  □ 交换机 #2 (数据面)
  □ 网线 × 25 根
```

### 第二步：接控制平面 (交换机 #1)

```
交换机 #1
┌─────────────────────────────────────┐
│ 口1 ●────── CS eth0                 │ 插上后 CS eth0 灯亮
│ 口2 ●────── CR-1 管理口             │ 查看: ip link show eth0 (CS)
│ 口3 ●────── CR-2 管理口             │       状态应为 UP
│ 口4 ●────── CR-3 管理口             │
│ 口5 ●────── CR-4 管理口             │
│ 口6 ●────── CR-5 管理口             │
│ 口7 ●────── CR-6 管理口             │
│ 口8 ●────── 笔记本 (抓包, 可选)      │
└─────────────────────────────────────┘
```

接线后验证：
```bash
# 在 CS 上
ping <CR-1 管理IP>
# 在笔记本上
ping <CS IP>
```

### 第三步：接数据平面 (交换机 #2)

```
交换机 #2
┌──────────────────────────────────────────┐
│ 核心面 (VLAN 10)                          │
│  口1  ●────── CR-1 核心口                 │
│  口2  ●────── CR-2 核心口                 │
│  口3  ●────── CR-3 核心口                 │
│  口4  ●────── CR-4 核心口                 │
│  口5  ●────── CR-5 核心口                 │
│  口6  ●────── CR-6 核心口                 │
│                                           │
│ 接入面1 (VLAN 20)                         │
│  口7  ●────── CR-1 接入口                 │
│  口8  ●────── AP-1 eth1                   │
│  口9  ●────── TS eth1                     │
│  口10 ●────── Host-1 (独立时)             │
│                                           │
│ 接入面2 (VLAN 30)                         │
│  口11 ●────── CR-2 接入口                 │
│  口12 ●────── AP-2 eth2                   │
│  口13 ●────── Host-2 (独立时)             │
│                                           │
│  口14 ●────── 笔记本 (镜像抓包)           │
└──────────────────────────────────────────┘
```

### 第四步：确认连接

```bash
# 在每台设备上检查对应网卡是否 UP
ip link show eth0    # CS — 应显示 state UP
ip link show eth1    # TS/AP-1 — 应显示 state UP
ip link show eth2    # AP-2 — 应显示 state UP

# 在笔记本上 ping 交换机所连设备
ping <CR-1 管理IP>
ping <CS_IP>
```

## 三、交换机配置

> 以下为通用配置逻辑，具体命令取决于交换机型号（中兴提供）。
> 与中兴对接时将此部分交给对方即可。

### 交换机 #1：控制平面（无需 VLAN）

```
目标: 所有端口 L2 互通

配置:
  - 创建 VLAN 1 (默认)
  - 端口 1~8 全部加入 VLAN 1, untagged
  - 端口 8 配置为镜像口 (mirror/SPAN):
     源端口: 1,2,3,4,5,6,7
     方向: both (收发都镜像)
```

### 交换机 #2：数据平面（VLAN + 端口隔离）

```
═══════════════════════════════════════
  VLAN 10 — 核心面 (CR↔CR)
═══════════════════════════════════════

目的: CR 之间通过 RID 格式互相通信

端口: 1,2,3,4,5,6 (CR1~6 核心口)
规则: 所有端口互通, untagged

═══════════════════════════════════════
  VLAN 20 — 接入面1 (CR-1 侧)
═══════════════════════════════════════

目的: CR-1 的接入侧设备通信, AID 格式

端口: 7 (CR-1接入), 8 (AP-1), 9 (TS), 10 (Host-1)
规则: untagged

端口隔离 (同一 VLAN 内分组):
  组 1: 端口 7 (CR-1接入) ←→ 端口 8 (AP-1)
  组 2: 端口 7 (CR-1接入) ←→ 端口 9 (TS)
  组 3: 端口 8 (AP-1) ←→ 端口 10 (Host-1)

效果:
  Host-1 ──只能通── AP-1 ──只能通── CR-1
  TS ──只能通── CR-1
  Host-1 不能直接通 TS (必须经过 CR-1)
  AP-1 不能直接通 TS

═══════════════════════════════════════
  VLAN 30 — 接入面2 (CR-2 侧)
═══════════════════════════════════════

目的: CR-2 的接入侧设备通信, AID 格式

端口: 11 (CR-2接入), 12 (AP-2), 13 (Host-2)
规则: untagged

端口隔离:
  组 1: 端口 11 (CR-2接入) ←→ 端口 12 (AP-2)
  组 2: 端口 12 (AP-2) ←→ 端口 13 (Host-2)

═══════════════════════════════════════
  镜像口
═══════════════════════════════════════

端口 14 配置为镜像口:
  源端口: 1~6 (核心面) + 7~13 (接入面)
  方向: both
  用途: 笔记本接此口, tcpdump 抓包分析
```

### VLAN 间通信

```
VLAN 10 ↔ VLAN 20:
  通过 CR-1 的核心口(口1) ↔ 接入口(口7) 完成
  不需要交换机做 VLAN 间路由

VLAN 10 ↔ VLAN 30:
  通过 CR-2 的核心口(口2) ↔ 接入口(口11) 完成

VLAN 20 和 VLAN 30:
  不能直接通信
  必须通过 VLAN 10 的 CR 转发 (标识网络路径)
```

## 四、验证连通性

### 4.1 L2 连通性 (控制面)

```bash
# CS → CR-1 管理口
sudo arping -I eth0 <CR-1管理口MAC>
# 预期: 收到 ARP 回复

# 笔记本抓包
tcpdump -i eth0 -c 5
# 预期: 看到 ARP 广播包
```

### 4.2 VLAN 隔离验证 (数据面)

```bash
# TS → CR-1 接入 (VLAN 20 内, 同组2)
sudo arping -I eth1 <CR-1接入MAC>
# 预期: 收到回复 ✅

# AP-1 → TS (VLAN 20 内, 不同组)
sudo arping -I eth2 <TS_MAC>   # 从AP-1的角度
# 预期: 没有回复 ❌ (端口隔离生效)

# AP-1 → AP-2 (跨 VLAN 20/30)
sudo arping -I eth1 <AP-2_MAC>
# 预期: 没有回复 ❌ (VLAN 隔离)
```

### 4.3 端口隔离验证

```
测试: Host-1 (端口10) → TS (端口9)

预期: 不通
原因: 端口10 只在隔离组3 (与端口8), 不在组2 (端口7↔9)
      Host-1 的帧到达交换机后被丢弃

验证: 在 Host-1 上 ping TS 的 IP
     → 不通 ✅ (隔离生效)
     → Host-1 只能通过 AP-1 → CR-1 → TS 的标识网络路径
```

## 五、网线汇总

| 交换机 | 已用端口 | 剩余 | 线缆类型 |
|--------|---------|------|---------|
| #1 控制面 | 7~8 | 16~24 | Cat5e/6 网线 |
| #2 数据面 | 13~14 | 10~34 | Cat5e/6 网线 |

每台 CR 需要 **3 根网线** (管理口 + 核心口 + 接入口)。
每台北交大设备需要 **1~2 根网线** (按网卡数)。
